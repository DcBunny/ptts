# 跳绳记录应用功能规划（第一版）

## 1. 目标与边界
- 目标：在用户选择时长后，记录小朋友跳绳次数，并保存本次训练视频。
- 当前阶段：先完成可运行的架构骨架和最小功能闭环（计时、计数、会话保存、视频路径预留）。
- 非目标（后续迭代）：自动动作识别算法、云端同步、家长多账号体系。

## 2. 输入 / 输出
- 输入：训练时长（30/60/120 秒）、开始/暂停/结束指令、跳绳次数事件（当前为手动模拟）。
- 输出：
  - 训练过程状态（倒计时、当前次数、录制状态）；
  - 训练结果（总次数、持续时长、时间戳、视频路径）；
  - 历史记录列表（本地存储）。

## 3. 功能模块拆分
1. 训练会话（Jump Session）
- 时长选择、开始、暂停、继续、结束。
- 会话期间实时更新剩余时间与跳绳次数。

2. 计数能力（Counter）
- 第一阶段：手动 +1 模拟计数事件。
- 后续可替换为摄像头识别事件流（保持接口不变）。

3. 视频录制（Recorder）
- 第一阶段：定义录制接口并保留视频路径字段。
- 后续接入 `camera` 插件实现真实录制。

4. 数据持久化（Repository + Storage）
- 会话结果写入本地（`shared_preferences`）。
- 支持读取历史会话。

## 4. 架构设计（MVVM + Riverpod）
- `lib/features/jump_session/view/`：页面渲染与交互。
- `lib/features/jump_session/view_model/`：状态机、计时器、业务编排。
- `lib/features/jump_session/data/`：模型、仓储、录制器抽象与实现。
- `lib/core/router/`：统一路由。
- `lib/core/storage/`：统一存储入口。
- `lib/core/i18n/`：统一文案读取入口。

## 5. 风险与假设
- 风险：真实视频录制与动作识别依赖设备权限和插件稳定性。
- 假设：当前迭代先保证架构可替换，允许录制器使用 Mock 实现。
- 验收标准：
  - 能选择时长并开始训练；
  - 可更新跳绳次数；
  - 到时或手动结束后保存一条会话记录；
  - 工程通过 `dart format .`、`flutter analyze`、`flutter test`。
