# AI 协作与 Skill 规范

## 1. AI 协作原则
- AI 输出默认视为“建议实现”，必须经人工 review。
- AI 修改代码后必须执行最小验证：`flutter analyze`。
- 涉及核心链路改动时，需补充或更新测试。

## 2. 需求表达规范
- 提需求时尽量给出：目标、约束、输入输出、验收标准。
- 涉及 UI 时优先补充设计稿、交互说明或示例截图。
- 涉及接口时提供字段定义、错误码、边界条件。

## 3. 变更记录
- AI 完成修改后，在 PR 描述中注明：
  - 变更文件范围。
  - 关键设计决策。
  - 已执行的验证命令与结果。

## 4. Skill 使用建议
- 可复用流程沉淀为 skill（如模型生成、脚手架、检查清单）。
- skill 文档需包含：适用场景、输入参数、执行步骤、失败处理。
- skill 改动需同步更新示例，避免文档与行为不一致。
