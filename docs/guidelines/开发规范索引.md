# 开发规范索引

## 1. 适用范围
- 本规范适用于 `tiaosheng` 项目所有开发成员与 AI 代理。
- 目标：统一工程结构、研发流程、代码质量与协作方式。
- 当规范与当前代码实现不一致时，以可运行代码为准，并在同次变更中更新文档。

## 2. 阅读顺序
1. 项目结构、目录组织、研发流程：`docs/guidelines/project_structure_and_workflow.md`
2. 代码风格、命名、注释、质量控制：`docs/guidelines/code_style_and_quality.md`
3. 架构与运行时（状态管理、路由、i18n、存储）：`docs/guidelines/architecture_and_runtime.md`
4. AI 协作方式：`docs/guidelines/ai_collaboration_and_skills.md`
5. 提交与 PR 规范：`docs/guidelines/contribution_and_pr.md`

## 3. 快速约束
- 功能模块统一按 `feature` 组织，禁止将业务代码直接堆到 `core`。
- 提交前必须执行：`flutter analyze`。
- 仓库存在测试时，提交前还必须执行：`flutter test`。
- 禁止手改生成文件：`*.g.dart`、`*.freezed.dart`。
- 新增规范优先补充到本目录，不分散在聊天记录或临时文档中。
